## 问题
扫描 3121 个文件时，进度条只显示 3000/3121，最后 121 个文件的进度没有实时更新。

## 解决方案
在后端扫描完成时，发送一次最终的实际进度事件，让前端显示准确的文件数量。

## 修改文件

### 1. `src-tauri/src/main.rs`
在扫描循环结束后，增加一次最终的进度发送：
```rust
// 扫描完成后，发送最终进度（确保显示实际数量）
let _ = app.emit("scan-progress", ScanProgress { 
    processed: processed_count, 
    total: current_total 
});
```

### 2. `src/App.tsx` (可选优化)
在扫描完成时，使用实际的文件数量更新进度：
```typescript
// 使用实际的文件数量更新最终进度
const actualFileCount = Object.values(result.files).filter(f => f.type === FileType.IMAGE).length;
updateTask(taskId, { 
    current: actualFileCount, 
    total: actualFileCount, 
    status: 'completed' 
});
```

## 预期效果
- 进度条会显示准确的 3121/3121
- 不会影响扫描速度
- 改动最小，风险低
