## 功能概述
在 SettingsModal 的储存栏（storage category）中添加主色调数据库管理功能，包括：
1. 数据库统计信息展示（总记录数、状态分布、文件大小）
2. 错误文件列表查看
3. 对错误文件重新生成主色调的功能

## 实现方案

### 1. 后端 Rust 新增命令
在 `src-tauri/src/color_db.rs` 中添加：
- `get_color_db_stats()` - 获取数据库统计信息
- `get_error_files()` - 获取所有错误状态的文件列表
- `retry_color_extraction(file_paths)` - 将指定文件重新加入待处理队列

### 2. 前端 API 桥接
在 `src/api/tauri-bridge.ts` 中添加对应的调用函数：
- `getColorDbStats()`
- `getErrorFiles()`
- `retryColorExtraction(filePaths)`

### 3. 设置面板 UI 扩展
在 `SettingsModal.tsx` 的 storage category 中添加新 section：
- **主色调数据库统计卡片**：显示总记录数、已提取/错误/待处理数量、数据库文件大小
- **错误文件管理区域**：
  - 显示错误文件数量
  - 展开后显示错误文件列表（文件路径、错误时间）
  - "重新生成"按钮（单个文件和全部重新生成）
  - 操作进度显示

### 4. 国际化支持
在 `translations.ts` 中添加相关翻译键：
- `settings.colorDbTitle`
- `settings.colorDbTotalRecords`
- `settings.colorDbExtracted`
- `settings.colorDbErrors`
- `settings.colorDbPending`
- `settings.colorDbFileSize`
- `settings.colorDbRetryErrors`
- `settings.colorDbRetryAll`
- `settings.colorDbRetrySuccess`

### 5. 样式设计
- 使用与现有性能监控面板类似的卡片式布局
- 错误文件列表使用可折叠设计
- 进度条和状态指示器与现有任务系统保持一致

## 文件修改清单
1. `src-tauri/src/color_db.rs` - 添加数据库查询函数
2. `src-tauri/src/main.rs` - 注册新的 Tauri 命令
3. `src/api/tauri-bridge.ts` - 添加前端 API 调用
4. `src/components/SettingsModal.tsx` - 添加 UI 组件
5. `src/utils/translations.ts` - 添加翻译文本

## 用户交互流程
1. 用户打开设置面板 → 储存栏
2. 自动加载主色调数据库统计信息
3. 如果有错误文件，显示警告提示
4. 用户可以：
   - 查看错误文件详情
   - 选择单个文件重新生成
   - 一键重新生成所有错误文件
5. 重新生成后，文件状态变为 pending，后台工作线程会自动处理
