## 问题
.aurora文件虽然保存了图片和布局信息，但切换根目录后无法读取，因为代码依赖外部库的`files`对象。

## 修复方案

### 1. 新增状态
```typescript
const [sessionFiles, setSessionFiles] = useState<Record<string, FileNode>>({});
```

### 2. 修改imageFiles useMemo
```typescript
const imageFiles = useMemo(() => {
  return internalSelectedIds
    .map(id => sessionFiles[id] || files[id])
    .filter(file => file && file.path);
}, [internalSelectedIds, files, sessionFiles]);
```

### 3. 修改handleLoadSession
- 从ZIP读取图片后创建Blob URL
- 为每个图片创建完整的FileNode对象存入sessionFiles
- 完整恢复manualLayouts（位置、大小、旋转）
- 恢复annotations（注释）
- 恢复zOrder（层级顺序）

### 4. 添加清理逻辑
组件卸载时释放Blob URL内存。

## 结果
.aurora文件将完全自包含，在任何库中都能完整恢复：
- 图片显示
- 位置、大小、旋转
- 层级顺序
- 注释信息
