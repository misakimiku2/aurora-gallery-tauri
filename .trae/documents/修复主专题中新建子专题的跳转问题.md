## 问题描述
当在主专题中点击"新建专题"创建子专题时，会错误地跳转到整个专题界面。而左侧栏"+"按钮创建根专题时跳转是正确的。

## 问题原因
在 `App.tsx` 的 `handleCreateTopic` 函数中，创建专题后**无条件**调用了 `handleNavigateTopics()` 进行跳转，导致无论是创建根专题还是子专题都会跳转。

## 修复方案
修改 `App.tsx` 第 2850-2886 行的 `handleCreateTopic` 函数，将无条件跳转改为条件跳转：

```typescript
// 修改前：
handleNavigateTopics();

// 修改后：
// 只有创建根专题（parentId === null）时才跳转到专题概览界面
// 创建子专题时保留在当前主专题中
if (parentId === null) {
    handleNavigateTopics();
}
```

## 预期行为
- 左侧栏"+"按钮创建根专题：跳转到专题概览界面 ✅
- 主专题中创建子专题：保留在当前主专题中 ✅
