## 问题分析
当前软件只有"记住文件夹"功能（针对单个文件夹保存布局设置），但缺少**全局默认布局设置**。当用户打开新文件夹时，总是使用硬编码的默认值（网格视图、按名称排序、不分组），导致用户设置瀑布流视图后，重启软件又恢复默认。

## 解决方案

### 1. 修改类型定义 (src/types.ts)
在 `AppSettings` 接口中添加全局默认布局设置字段：
```typescript
export interface AppSettings {
  // ... 现有字段
  defaultLayoutSettings: {
    layoutMode: LayoutMode;
    sortBy: SortOption;
    sortDirection: SortDirection;
    groupBy: GroupByOption;
  };
}
```

### 2. 修改常量默认值 (src/constants.ts)
添加默认布局设置的常量。

### 3. 修改 App.tsx
- 在初始状态中添加 `defaultLayoutSettings` 的默认值
- 在初始化时从保存的数据中恢复全局布局设置
- 当用户更改布局/排序/分组时，如果没有针对当前文件夹保存特定设置，则更新全局默认设置
- 或者添加一个"设为默认"按钮让用户主动设置

### 4. 修改设置界面 (src/components/SettingsModal.tsx)
添加全局布局设置的可视化配置界面，让用户可以：
- 选择默认视图模式（网格/自适应/列表/瀑布流）
- 选择默认排序方式（名称/日期/大小）
- 选择默认排序方向（升序/降序）
- 选择默认分组方式（无/类型/日期/大小）

### 5. 应用全局设置
当打开新文件夹时，使用全局默认设置而不是硬编码值。

## 实现步骤
1. 在 `types.ts` 的 `AppSettings` 中添加 `defaultLayoutSettings` 字段
2. 在 `constants.ts` 中添加默认布局设置常量
3. 在 `App.tsx` 中：
   - 更新初始状态
   - 在初始化时加载全局设置
   - 修改新标签页/新文件夹的默认设置逻辑
4. 在 `SettingsModal.tsx` 中添加全局布局设置UI
5. 确保设置自动保存到用户数据

这个方案可以让用户设置一次全局偏好，所有新打开的文件夹都会使用这些设置，同时仍然保留"记住文件夹"功能来覆盖特定文件夹的设置。
